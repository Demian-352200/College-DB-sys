#核心功能：面向用户的高校数据增删改查

##需求描述：
本系统提供分级权限管理的数据操作功能，支持不同用户角色对高校数据进行增删改查操作。

##用户角色定义：
  普通用户：可查询所有数据、添加新高校数据（需审核）、添加评价。
  管理员：拥有所有权限，包括数据删除、修改和审核。

##功能模块设计：

1. 数据查询模块
功能描述：支持所有用户自由查询高校数据
功能点：
    高校基本信息查询（名称、省份、类型等）
    多条件组合筛选（985/211/双一流、省份、类型等）
    饮食偏好查询
    评价信息浏览
    地理空间查询（按地图区域筛选）
    关键词搜索（高校名称、特点等）
权限控制：对所有用户开放

2. 数据添加模块
功能描述：支持用户添加新数据
功能点：
    普通用户添加新高校（进入审核流程）
    普通用户添加高校评价（直接发布）
    管理员直接添加高校（无需审核）
    数据验证（防止重复添加、格式检查）
权限控制：
    添加高校：普通用户（需审核）、管理员（直接发布）
    添加评价：所有认证用户

3. 数据修改模块
功能描述：支持用户修改已有数据
功能点：
    普通用户修改自己添加的评价
    管理员修改任何高校基本信息
    管理员修改饮食偏好信息
    修改记录追踪（记录修改人和修改时间）
权限控制：
    修改评价：仅评价发布者
    修改高校数据：仅管理员

4. 数据删除模块
功能描述：支持删除错误数据
功能点：
    管理员删除高校数据（连带删除相关评价）
    管理员删除不当评价
    普通用户删除自己添加的评价
    删除确认机制（防止误操作）
权限控制：
    删除高校：仅管理员
    删除评价：评价发布者或管理员

5. 审核管理模块
功能描述：管理普通用户提交的数据
功能点：
    待审核高校数据列表
    审核通过/拒绝操作
    审核意见填写
    通知用户审核结果
权限控制：仅管理员

##关键业务流程：
1. 普通用户添加高校流程

    用户填写高校基本信息表单

    系统验证数据完整性

    数据存入待审核状态

    管理员收到审核通知

    管理员审核通过/拒绝

    用户收到审核结果通知

2. 评价管理流程

    用户选择高校并添加评价

    评价立即发布

    评价作者可修改或删除自己的评价

    管理员可修改或删除任何评价

3. 数据修改流程

    用户/管理员请求修改数据

    系统验证操作权限

    记录修改历史（谁、何时、修改了什么）

    更新数据

##异常处理设计
    1. 权限不足：提示"无操作权限"
    2. 数据不存在：提示"数据不存在或已被删除"
    3. 重复添加：提示"数据已存在"
    4. 审核中数据：提示"数据审核中，暂不可修改"
    5. 网络异常：提供重试机制
    6. 其他
